136 string fullHelpMessage(void)
137  {
138  string msg =
139 "\n"
140 "SYNOPSIS\n"
141 "\n"
142 "Compute the distribution of a membrane physical property about a membrane\n"
143 "protein.\n"
144 "\n"
145 "DESCRIPTION\n"
146 "The purpose of this tool is to compute one of a number of physical \n"
147 "properties on a 2D grid surrounding a membrane protein. The properties \n"
148 "number density, height, molecular order parameter, and orientation vector,\n"
149 "although the code is written to make it easy to add other quantities (see \n"
150 "below). The system is aligned against the coordinates of the selection \n"
151 "specified with --align-selection using the first unskipped frame as \n"
152 "reference, unless the --ref-structure option is given, in which case \n"
153 "the file specified there is used (--align-selection is still applied)\n"
154 "The alignment is performed in two dimensions, so that the \n"
155 "lipid bilayer is not tilted or shifted; it is assumed that the bilayer\n"
156 "normal is the z-axis, and that the bilayer center is at z=0\n"
157 "\n"
158 "By default, the target is treated as a single large entity. The --splitby\n"
159 "flag will let you break it up into individual molecules, segments, or \n"
160 "residues.\n"
161 "\n"
162 "Options\n"
163 "--calc The type of calculation to be performed.\n"
164 " density: number density of the selection\n"
165 " height: average z-position of the centroid of the selection\n"
166 " order: molecular order parameter (see below)\n"
167 " vector: orientation vector\n"
168 " dipole: dipole vector\n"
169 "\n"
170 " The molecular order parameter is calculated using the \n"
171 " principal axes of the selection; the 2nd and 3rd axes are\n"
172 " averaged, and plugged into the standard 0.5 (3 cos^2 - 1) \n"
173 " formula, relative the z-axis. If the selection is a lipid\n"
174 " chain, then the values are comparable to the ones seen for\n"
175 " lipid order parameters.\n"
176 "\n"
177 " The orientation vector is average of the xy components of the\n"
178 " first principal axis of the selection. Unlike the other \n"
179 " options, which return scalars, this returns a 2D vector, \n"
180 " which can be plotted in gnuplot using the \"with vector\" \n"
181 " option.\n"
182 "\n"
183 " Dipole is the electric dipole, computed using partial\n"
184 " charges, which must be present for this calculation to work.\n"
185 "\n"
186 "\n"
187 "\n"
188 "EXAMPLE\n"
189 "\n"
190 "membrane_map --xmin -30 --xmax 30 --ymin -30 --ymax 30 --xbins 30 --ybins 30 --splitby mol example.psf dark_ensemble_20.dcd --align-selection 'segid == \"RHOD\"' --target-selection 'resname == \"DHA\"'\n"
191 "\n"
192 " This sets the histograms to run from -30:30 in x and y, with \n"
193 " 2 ang x 2 ang bins. It uses the segment name RHOD to align the\n"
194 " snapshots, and uses DHA chains as the targets. Since no \n"
195 " calculation type is specified, a number density is calculated. \n"
196 " The DHA chains are split up on the basis of connectivity.\n"
197 "\n"
198 "If you wish to examine membrane properties in general (e.g. for a phase-\n"
199 "separated membrane with no protein) you can choose to not use an alignment\n"
200 "selection. However, since domains may drift around during the simulation, \n"
201 "you may want to run the code on discrete ranges of frames rather than just \n"
202 "averaging over the whole trajectory. For example, you could modify the \n"
203 "previous example to be:\n"
204 "\n"
205 "membrane_map --range 200:299 --xmin -30 --xmax 30 --ymin -30 --ymax 30 --xbins 30 --ybins 30 --splitby mol example.psf dark_ensemble_20.dcd --target-selection 'resname == \"DHA\"'\n"
206 "\n"
207 "This calculation would not perform any alignment, and would skip the first\n"
208 "200 frames, use the next 100 frames, then skip the rest of the trajectory.\n"
209 "\n"
210 "POTENTIAL COMPLICATIONS\n"
211 "\n"
212 "The code will break if the alignment and target selections overlap, \n"
213 "because the 2D alignment works by setting the z-coordinates of the \n"
214 "alignment selection to 0. \n"
215 "\n"
216 "In regions where there's no data (e.g. inside the region occluded by \n"
217 "whatever you're aligning to), the code outputs a value of -999999999.\n"
218 "chosen as the ANSI standard insane value. The exception is if you're\n"
219 "doing a density calculation, in which case there's no divide by zero\n"
220 "and sanity reigns everywhere.\n"
221 "\n"
222 "The options --upper-only and --lower-only let you calculate properties\n"
223 "using only the upper and lower leaflets respectively. The check is done\n"
224 "for each frame, so these options will handle the case where a component\n"
225 "is capable of flipping between leaflets on the MD timescale. However,\n"
226 "the implementation assumes that the membrane has been previously centered\n"
227 "at z=0. For obvious reasons, you can't specify both --upper-only and \n"
228 "--lower-only at the same time. Note: the stated number of matching target\n"
229 "molecules output at the beginning of the run does not take this restriction\n"
230 "into account.\n"
231 "\n"
232 "IMPLEMENTING NEW QUANTITIES\n"
233 "\n"
234 "Implementing new quantities is quite easy as long as they return either\n"
235 "a scalar (double) or a vector (GCoord). You begin by editing \n"
236 "membrane_map.hpp -- all of the classes that do the work are subclasses of \n"
237 "CalcProperty, so you'll need to create a new class analogous to the ones\n"
238 "already there, e.g. CalcDensity or CalcMolOrder. You'll need to supply\n"
239 "a constructor that calls the CalcProperty constructor and a method called\n"
240 "calc that will do the calculation. The last step of the calc method must\n"
241 "be calling the incr function, which will add the value into the histogram.\n"
242 "If the quantity being calculated is a density, you may need to supply a \n"
243 "normalize method as well (see CalcDensity for an example).\n"
244 "\n"
245 "Once the class is written, you just have to hook it in so that the binary\n"
246 "knows about it. To do this, edit membrane_map.cpp. First, you'll need to\n"
247 "add the new type to the enum CalcType (near the top of membrane_map.cpp).\n"
248 "Second, edit the postConditions method in the ToolOptions class to add \n"
249 "your calculation type (it's just a string of if-else-ifs). Third, \n"
250 "edit the switch statement in the main body of the code to add your \n"
251 "new method for calculating (there's only 1 switch). Finally, \n"
252 "if you intend anyone else to use your method, edit the documentation \n"
253 "string in addGeneric (also in ToolOptions) and the fullHelpMessage \n"
254 "function.\n"
255 "\n"
256  "\n";
257  return(msg);

263 int main(int argc, char *argv[])
264  {
265 
266  string hdr = invocationHeader(argc, argv);
267  cout << "# " << hdr << endl;
268 
269  opts::BasicOptions* bopts = new opts::BasicOptions(fullHelpMessage());
270  opts::TrajectoryWithFrameIndices* tropts = new opts::TrajectoryWithFrameIndices;
271  opts::BasicSplitBy *sopts = new opts::BasicSplitBy;
272  opts::RequiredArguments* ropts = new opts::RequiredArguments;
273  ToolOptions* topts = new ToolOptions;
274 
275  opts::AggregateOptions options;
276  options.add(bopts).add(tropts).add(ropts).add(topts).add(sopts);
277  if (!options.parse(argc, argv))
278  {
279  exit(-1);
280  }
281 
282  AtomicGroup system = tropts->model;
283  pTraj traj = tropts->trajectory;
284  vector<uint> frames = tropts->frameList();
285  traj->readFrame(frames[0]);
286  traj->updateGroupCoords(system);
287 
288  AtomicGroup align_to;
289  AtomicGroup reference;
290  if (topts->has_align)
291  {
292  align_to = selectAtoms(system, topts->align_selection);
293  if ((topts->reference_filename).length() > 0)
294  {
295  AtomicGroup reference_system = createSystem(topts->reference_filename);
296  reference = selectAtoms(reference_system, topts->align_selection);
297  }
298  else
299  {
300  reference = align_to.copy();
301  }
302  }
303 
304  AtomicGroup apply_to = selectAtoms(system, topts->target_selection);
305 
306  vector<AtomicGroup> targets = sopts->split(apply_to);
307  cout << "# Found " << targets.size() << " matching molecules" << endl;
308 
309  // Set up storage for our property.
310  double xmin = topts->xmin;
311  double xmax = topts->xmax;
312  double ymin = topts->ymin;
313  double ymax = topts->ymax;
314  uint xbins = topts->xbins;
315  uint ybins = topts->ybins;
316  double xwidth = (xmax - xmin)/xbins;
317  double ywidth = (ymax - ymin)/ybins;
318 
319  CalcPropertyBase *calculator;
320  switch (topts->type)
321  {
322  case DENSITY:
323  calculator = new CalcDensity(xbins, ybins, xwidth, ywidth);
324  break;
325  case ORDER:
326  calculator = new CalcMolOrder(xbins, ybins);
327  break;
328  case HEIGHT:
329  calculator = new CalcHeight(xbins, ybins);
330  break;
331  case VECTOR:
332  calculator = new CalcOrientVector(xbins, ybins);
333  break;
334  case DIPOLE:
335  calculator = new CalcDipole(xbins, ybins);
336  break;
337  default: // this can't happen, set in option handling
338  cerr << "ERROR: unknown calculation type" << endl;
339  exit(-1);
340  }
341 
342  // If we're doing dipole, we need charges
343  if (topts->type == DIPOLE)
344  {
345  // check the first atom to see if charges were set
346  if (!system[0]->checkProperty(Atom::chargebit))
347  {
348  cerr << "Dipole calculation require a model with charges"
349  << endl;
350  cerr << "Exiting..." << endl;
351  exit(-1);
352  }
353 
354  }
355 
356  // We don't want the transformation to tilt the membrane, so we'll
357  // zero out the z coordinates before using the alignment.
358  // We'll do the same
359  for (AtomicGroup::iterator i = reference.begin();
360  i!= reference.end();
361  ++i)
362  {
363  (*i)->coords().z() = 0.0;
364  }
365 
366  // loop over frames in the trajectory
367  for (uint i=0; i<frames.size(); ++i)
368  {
369  traj->readFrame(frames[i]);
370  traj->updateGroupCoords(system);
371 
372  if (topts->has_align)
373  {
374  // zero out the alignment selections z-coordinate
375  AtomicGroup align_to_flattened = align_to.copy();
376  for (AtomicGroup::iterator j = align_to_flattened.begin();
377  j!= align_to_flattened.end();
378  ++j)
379  {
380  (*j)->coords().z() = 0.0;
381  }
382 
383 
384  // get the alignment matrix
385  GMatrix M = align_to_flattened.superposition(reference);
386  M(2,2) = 1.0; // Fix a problem caused by zapping the z-coords...
387  XForm W(M);
388 
389  // align the stuff we're goign to do the calculation on
390  apply_to.applyTransform(W);
391  }
392 
393  // Calculate something
394  uint xbin, ybin;
395  for (vector<AtomicGroup>::iterator j = targets.begin();
396  j != targets.end();
397  ++j)
398  {
399  GCoord centroid = j->centroid();
400  // Skip molecules outside the xy range of interest
401  if ( (centroid.x() < xmin) || (centroid.x() > xmax) ||
402  (centroid.y() < ymin) || (centroid.y() > ymax)
403  )
404  {
405  continue;
406  }
407  // If the user chose to look at only one leaflet,
408  // skip molecules in the opposite leaflet.
409  // Note: this assumes that the membrane is centered at z=0
410  else if ((centroid.z() > 0) && topts->lower_only)
411  {
412  continue;
413  }
414  else if ((centroid.z() < 0) && topts->upper_only)
415  {
416  continue;
417  }
418  else
419  {
420  xbin = (uint)(centroid.x() - xmin) / xwidth;
421  ybin = (uint)(centroid.y() - ymin) / ywidth;
422  }
423 
424  // do the work
425  calculator->calc(*j, xbin, ybin);
426  }
427 
428  }
429 
430  // loop over bins and dump out the values
431  calculator->normalize(frames.size());
432 
433  cout << "# X\tY\tValue(s)" << endl;
434  for (uint i = 0; i < xbins; ++i)
435  {
436  double xval = xmin + xwidth*i;
437  for (uint j = 0; j < ybins; ++j)
438  {
439  double yval = ymin + ywidth*j;
440 
441  cout << xval << "\t"
442  << yval << "\t"
443  << calculator->print(i,j)
444  << endl;
445  }
446  cout << endl;
447  }
448  }
