157  "\n"

159 "supported by LOOS and efficiently produces a merged trajectory in\n"
160 "DCD format. It can also produce a second, downsampled trajectory,\n"
161 "and can recenter and reimage the coordinates at the same time.\n"

163 "Unlike other tools, such as catdcd, merge-traj works by appending to\n"
164 "existing trajectory files instead of rewriting them from scratch each\n"
165 "time. This can dramatically improve the performance in a common usage\n"

168 "available to date. \n"
169 "\n"
170 "The user specifies the target for merged trajectory, and a list of\n"
171 "trajectory files to be merged. The program determines the number of \n"

173 "of trajectories to be merged, skipping that number of frames and only\n"
174 "then beginning to append new frames. This means that a) the user\n"
175 "must specify the trajectories in the correct order, and b) all \n"
176 "trajectories must be specified each time (not just the newest files).\n"
177 "merge-traj correctly handles the case where one of the trajectories \n"
178 "to be merged has grown since the previous merge.\n"

180 "Options related to downsampling\n"
181 "\n"
182 "--downsample-dcd a second merged DCD file, with frames written at\n"

184 "--downsample-rate integer specifying how often to write to the \n"
185 " downsampled DCD file, e.g. 10 means write every\n"
186 " 10th frame\n"
187 "Note: the downsampled DCD file must be synchronized with the fully sampled\n"
188 "one. This is the user's responsibility, as the code doesn't do any \n"
189 "additional checking. The easiest way is to put the command line into\n"
190 "a script to ensure that both files are always used.\n"
191 "\n"
192 "Options related to recentering\n"
193 "\n"
194 "It is often convenient to clean up the trajectory at merge time, removing\n"
195 "center of mass motion for some component of the system (e.g. the protein).\n"
196 "Accordingly, merge-traj has the following options\n"
197 "\n"
198 " --centering-selection the centroid of the atoms specificed by the \n"
199 " selection string will be moved to the origin in\n"
200 " each frame. No rotations are performed.\n"
201 " --xy-centering-selection same as --centering-selection, except only move in\n"
202 " the xy plane. Can't be used with --centering-selection\n"
203 " but can be combined with --z-centering-selection\n"
204 " --z-centering-selection same as --centering-selection, except only move in\n"
205 " the z direction. Can't be used with --centering-selection\n"
206 " but can be combined with --xy-centering-selection\n"
207 " --selection-is-split This flag indicates that the selection specified\n"
208 " by --centering-selection may be split across image\n"
209 " boundaries, in which case the centroid can be far\n"
210 " from where the atoms are actually located. In \n"
211 " this case, the recentering is performed in 2 \n"
212 " stages, first putting the selection into a \n"
213 " single image, then recentering. Works correctly with\n"
214 " all 3 centering variants\n"
215 " --fix-imaging Ensure that molecules are not broken across \n"
216 " image boundaries. This is generally necessary\n"

219 " performs a final centering and reimaging operation\n"
220 " using this selection. The idea is that for \n"
221 " really messy groups, you might need to center and\n"
222 " reimage multiple ways to get everything to work.\n"
223 " Handles many of the same cases as \n"
224 " --selection-is-split, so you can try either to see\n"
225 " which works for you.\n"
226 " --postcenter-xy like --postcenter, but only the xy plane\n"
227 " --postcenter-z like --postcenter, but only the z-axis\n"
228 "\n"
229 "\n"
230 "In addition, for merging GROMACS XTC files there is an additional flag:\n"
231 "\n"
232 "--skip-first-frame XTC files can contain the initial structure as\n"

235 "\n"
236 "\n"
237 "EXAMPLE\n"

239 "\n"
240 "Here is an example command line:\n"
241 "\n"
242 "merge-traj --centering-selection 'segid==\"OPSN\"' --downsample-dcd merged_1ns.dcd \\\n"
243 " --downsample-rate 10 start.psf merged.dcd traj.[0-9].dcd \\\n"
244 " traj.[1-9][0-9].dcd traj.[1-9][0-9][0-9].dcd\n"
245 "\n"
246 "This will merge a set of trajectory files named traj.0.dcd, traj.1.dcd, \n"
247 "etc., going up to hundreds of trajectory files as necessary (this is \n"
248 "tcsh, but bash would be similar). It's necessary to specify the merge \n"
249 "this way in order to get the files in the proper order on the command \n"

265 "The rationale for having --xy-centering-selection and z-centering selection\n"

277 "--selection-is-split. The trick in that case is to use different selections\n"
278 "for centering and postcentering, e.g. a single lipid molecule for the initial\n"
279 "centering (which will ensure the bilayer is now largely intact), followed \n"

293 struct ScanfFmt
294 {
295  ScanfFmt(const string& s) : fmt(s) { }
296 
297  uint operator()(const string& s) const
298  {
299  uint d;
300  if (sscanf(s.c_str(), fmt.c_str(), &d) != 1)
301  {
302  cerr << boost::format("Bad conversion of '%s' using format '%s'\n") % s % fmt;
303  exit(-20);
304  }
305 
306  return(d);
307  }
308 
309  string fmt;
310 };
311 
312 
313 
314 struct RegexFmt
315 {
316  RegexFmt(const string& s) : fmt(s)
317  {
318  regexp = boost::regex(s, boost::regex::perl);
319  }
320 
321  uint operator()(const string& s) const
322  {
323  boost::smatch what;
324  if (boost::regex_search(s, what, regexp))
325  {
326  for (uint i=0; i<what.size(); ++i)
327  {
328  string submatch(what.str(i));
329  char *p;
330  if (submatch.empty()) // Should never happen?
331  continue;
332 
333  uint val = strtoul(submatch.c_str(), &p, 10);
334  if (*p == '\0')
335  return(val);
336  }
337  }
338 
339  cerr << boost::format("Bad conversion of '%s' using regexp '%s'\n") % s % fmt;
340  exit(-20);
341  }
342 
343  string fmt;
344  boost::regex regexp;
345 };

368 // Given a vector of trajectory filenames, along with a functor for
369 // extracting the frame index from the filename, sorts it in numeric
370 // ascending order...
371 
372 template<class FmtOp>
373 vector<string> sortNamesByFormat(vector<string>& names, const FmtOp& op)
374 {
375  uint n = names.size();
376  vector<SortDatum> bound(n);
377  for (uint i=0; i<n; ++i)
378  {
379  bound[i] = SortDatum(names[i], op(names[i]));
380  }
381 
382  sort(bound.begin(), bound.end());
383 
384  vector<string> sorted(n);
385  for (uint i=0; i<n; ++i)
386  sorted[i] = bound[i].s;
387 
388  return(sorted);
389 }

393 int main(int argc, char *argv[])
394 {
395  string hdr = invocationHeader(argc, argv);
396  opts::BasicOptions* bopts = new opts::BasicOptions(fullHelpMessage());
397  ToolOptions* topts = new ToolOptions;
398  opts::RequiredArguments* ropts = new opts::RequiredArguments;
399  ropts->addArgument("model", "model-filename");
400  ropts->addArgument("output_traj", "output-trajectory");
401  ropts->addVariableArguments("input_traj", "trajectory");
402 
403  opts::AggregateOptions options;
404  options.add(bopts).add(topts).add(ropts);
405  if (!options.parse(argc, argv))
406  exit(-1);
407 
408  model_name = ropts->value("model");
409  output_traj = ropts->value("output_traj");
410  input_dcd_list = ropts->variableValues("input_traj");
411 
412  if (topts->sort_flag)
413  {
414  if (!topts->scanf_spec.empty())
415  {
416  input_dcd_list = sortNamesByFormat(input_dcd_list, ScanfFmt(topts->scanf_spec));
417  }
418  else
419  {
420  input_dcd_list = sortNamesByFormat(input_dcd_list, RegexFmt(topts->regex_spec));
421  }
422  }
423 
424  cout << hdr << endl;
425  AtomicGroup system = createSystem(model_name);
426 
427  // We check for specifying both xy/z and full in the code
428  // that processes the command line options, so we don't
429  // have to do it here
430  bool full_recenter = false;
431  bool xy_recenter = false;
432  bool z_recenter = false;
433  bool post_recenter = false;
434  bool xy_post_recenter = false;
435  bool z_post_recenter = false;
436  if ( center_selection.length() != 0 )
437  {
438  full_recenter = true;
439  }
440  if ( xy_center_selection.length() != 0 )
441  {
442  xy_recenter = true;
443  }
444  if ( z_center_selection.length() != 0 )
445  {
446  z_recenter = true;
447  }
448  if (postcenter_selection.length() != 0)
449  {
450  post_recenter = true;
451  }
452  if (postcenter_xy_selection.length() != 0)
453  {
454  xy_post_recenter = true;
455  }
456  if (postcenter_z_selection.length() != 0)
457  {
458  z_post_recenter = true;
459  }
460 
461  pTrajectoryWriter output = createOutputTrajectory(output_traj, true);
462 
463  pTrajectoryWriter output_downsample;
464  bool do_downsample = (output_traj_downsample.length() > 0);
465  if (do_downsample)
466  {
467  output_downsample = createOutputTrajectory(output_traj_downsample, true);
468  }
469 
470  // Set up to do the recentering
471  vector<AtomicGroup> molecules;
472  AtomicGroup center, xy_center, z_center;
473  AtomicGroup post_center, xy_post_center, z_post_center;
474  vector<AtomicGroup>::iterator m;
475  if ( full_recenter )
476  {
477  center = selectAtoms(system, center_selection);
478  }
479  else
480  {
481  if ( xy_recenter )
482  {
483  xy_center = selectAtoms(system, xy_center_selection);
484  }
485  if ( z_recenter )
486  {
487  z_center = selectAtoms(system, z_center_selection);
488  }
489  }
490 
491  if ( full_recenter || xy_recenter || z_recenter || reimage_by_molecule )
492  {
493  if ( system.hasBonds() )
494  {
495  molecules = system.splitByMolecule();
496  }
497  else
498  {
499  molecules = system.splitByUniqueSegid();
500  }
501  }
502 
503  if (post_recenter)
504  {
505  post_center = selectAtoms(system, postcenter_selection);
506  }
507  else
508  {
509  if ( xy_post_recenter )
510  {
511  xy_post_center = selectAtoms(system, postcenter_xy_selection);
512  }
513  else if ( z_post_recenter )
514  {
515  z_post_center = selectAtoms(system, postcenter_z_selection);
516  }
517  }
518 
519  uint original_num_frames = output->framesWritten();
520  cout << "Target trajectory "
521  << output_traj
522  << " has "
523  << original_num_frames
524  << " frames."
525  << endl;
526 
527  uint previous_frames = 0;
528  vector<string>::iterator f;
529  for (f=input_dcd_list.begin(); f!=input_dcd_list.end(); ++f)
530  {
531  pTraj traj=createTrajectory(*f, system);
532  int nframes = traj->nframes();
533  if (skip_first_frame && nframes > 1)
534  {
535  nframes--;
536  }
537  cout << "File: " << *f << ": " << nframes;
538 
539  if ( previous_frames + nframes <= original_num_frames)
540  // all of this file is contained in the existing file, skip it
541  {
542  // increment the frame pointer
543  previous_frames += nframes;
544  cout << " ( " << previous_frames << " )"
545  << "\tSkipping trajectory "
546  << endl;
547 
548  }
549  else
550  // we need at least some of the data from this file
551  {
552  int frames_to_skip = original_num_frames - previous_frames;
553  if ( frames_to_skip > 0 )
554  {
555  traj->seekFrame(frames_to_skip-1);
556  }
557  else
558  {
559  frames_to_skip = 0;
560  }
561 
562  previous_frames += frames_to_skip;
563 
564  // if this is an xtc file, we need to skip 1 more frame
565  if (skip_first_frame)
566  {
567  traj->readFrame();
568  }
569 
570  cout << " ( " << previous_frames + nframes - frames_to_skip
571  << " ) "
572  << "\t Writing " << nframes - frames_to_skip
573  << " frames."
574  << endl;
575 
576  while ( traj->readFrame() )
577  {
578  traj->updateGroupCoords(system);
579 
580  // Find the smallest box dimension
581  GCoord box = system.periodicBox();
582  double smallest=1e20;
583  for (int i=0; i<3; i++)
584  {
585  if (box[i] < smallest)
586  {
587  smallest = box[i];
588  }
589  }
590 
591  smallest /=2.0;
592 
593 
594  // If molecules can be broken across image bondaries
595  // (eg GROMACS), then we may need 2 translations to
596  // fix them -- first, translate the whole molecule such
597  // that a single atom is at the origin, reimage the
598  // molecule, and put it back
599  if (reimage_by_molecule)
600  {
601  for (m=molecules.begin(); m != molecules.end(); ++m )
602  {
603  // This is relatively slow, so we'll skip the
604  // cases we know we won't need this -- 1 particle
605  // molecules and molecules with small radii
606  // Note: radius(true) computes the max distance between atom 0
607  // and all other atoms in the group. In certain perverse
608  // cases the centroid can be closer than 1/2 box to all atoms
609  // even when the molecule is split.
610  if ( (m->size() > 1) && (m->radius(true) > smallest) )
611  {
612  m->mergeImage();
613  m->reimage();
614  }
615  }
616  }
617 
618 
619  if ( full_recenter || xy_recenter || z_recenter)
620  {
621  // If the selection is split, then we effectively need to
622  // do the centering twice. First, we pick one atom from the
623  // centering selection, translate the entire system so it's
624  // at the origin, and reimage. This will get the selection
625  // region to not be split on the image boundary. At that
626  // point, we can just do regular imaging.
627  if (selection_split)
628  {
629  GCoord centroid;
630  if (full_recenter)
631  {
632  centroid = center[0]->coords();
633  }
634  else
635  {
636  if (xy_recenter)
637  {
638  centroid.x() = xy_center[0]->coords().x();
639  centroid.y() = xy_center[0]->coords().y();
640  }
641  if (z_recenter)
642  {
643  centroid.z() = z_center[0]->coords().z();
644  }
645  }
646 
647  system.translate(-centroid);
648 
649  for (m=molecules.begin(); m!=molecules.end(); m++)
650  {
651  m->reimage();
652  }
