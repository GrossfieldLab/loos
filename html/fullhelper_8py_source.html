7 This tool helps you wrap fullhelp messages for loos C++ source so that
8 they will be formatted correctly in the WYSIWYG loos style, where all the
9 needed punctuation and whitespace are specified in source. The objective is to
10 provide a way for users to write their desired fullhelp message in softwrapping
11 text editors, fairly common these days, as a plain text file. The correct
12 wrapping and line-decorating are then done by this script, and the adjusted
13 text is then written into the source at a spot specified by a locator or signal
14 string. This is done by assembling the entire updated source code as a list,
15 then substituting it into a copy of the original source code (in case something
16 bad happens). You can check this file, which will have the same name as your
17 tool's source but prefixed with 'fullhelp-test', and if you like what you see
18 renaming the fullhelp-test file to be the source file. The only difference
19 should be the replacement of your signal string with the fullhelp string.
20 
21 The intention of this script is for you to write your source code as you would,
22 including a placeholder string instead of your fullhelpstring in the definition
23 of 'string fullHelpMessage(void)' such as:
24 string fullHelpMessage(void) {
25  string msg =
26  "XXX";
27 
28  return(msg);
29 }
30 
31 In this way you can continue to work on your code, able to build it and even
32 test the appearance of the options framework help phrases, without having to
33 come up with and correctly format a handsome and useful full help until the
34 end. Obviously you can use any string you want, but make good choices. This
35 script will find and replace only the first instance of the signal string.
36 You need to include its surrounding quotes (easily done in bash by
37 including the signal string inside of doublequotes nested in single quotes, see
38 usage string). You can also make tweaks to the fullhelp source (the soft-wrapped
39 text file), then see what they look like by up-arrow rerunning your fullhelper
40 command at your terminal repeatedly.
41 
42 The three obligatory arguments are the name of .txt containing your full help
43 message, the source code file you'd like to sub your message into, and the
44 locator string you are using as a target for replacement by the full help
45 message. The fourth, optional, argument is the column count to wrap to. The
46 default column count is 80.
47 """
